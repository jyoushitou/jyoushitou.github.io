网络编程

字节序处理和消息队列控制

序列化

  • 使用protobuf等序列化成字符串

大端序

  • 指数据从高地址开始存储

小端序

  • 指数据从低地址开始存储

字节序问题

  • 为避免通讯过程的编码不同
  • 字节序使用大端序，即网络字节序

通过函数判断 

bool is_bug_endian()
{
    int num =1;
    if(*(char*)&num==1)
    {
        return 0;
    }
    else
    {
        return 1;
    }
}
  • 0(flase)为小端，1(ture)为大端

asio提供的

host_to_network_long()

  • 网络转换成long类型

host_to_network_short()

  • 网络转换成short类型

MySQL

事务

  • 一组操作的集合，所有指令顺序执行，有一条操作失败则退回到事务执行前的状态

SELECT @@autocommit;

  • 查看提交方式
  • 1为自动
  • 0为手动

set @@autocommit-0;

  • 设置手动提交
  • 之后所有的操作临时执行

commit;

  • 提交事务，让操作保存与数据与表
  • 保证事务是一起的执行的

rollback

  • 回滚操作
  • 事务操作失败后使用这个回滚

start transaction;

  • 开启事务
  • 即开启局部手动的提交事务

四大特性

原子性ACID

  • 不可分割最小单元

一致性

  • 事务完成时，必须使所有的数据保存一致状态

隔离性

  • 数据库系统提供的隔离机制，保证事务在不受外部并发操作影响的独立环境下运行

持久性

  • 事务一旦提交或回滚 ，他对数据库的中的数据改变是永久的

并发事务问题

脏读

  • 一个事务读到另外一个事务未提交的数据

不可重复读

  • 一个事务先后到读取同一条记录，但两次读取的数据不同，称之为不可重复读

幻读

  • 一个事务按照条件查询数据是，没有对应的数据行，但是插入数据时，又发现这行数据已经存在，好像出现了幻影

解决方法：事务隔离级别

  • 级别越高效率越低，安全性越低
Read uncommitted 读未提交
  • 啥也不解决
Read commited 读已提交
  • 解决幻读
Repeatable Read 可重复读
  • 解决不可重复读，幻读
  • 默认事务隔离级别
  • orcal
Serializabele 串行化
  • 解决脏读，不可重复读，幻读
查看事务隔离级别 
SELECT @@TRANSACTION_ISOLATION;
设置当前窗口的隔离·级别 
SET [SEESSION|GLOBAL] TRANSACTION ISOLATION LEVEL {${级别}};
