# 网络编程

## 粘包问题

### 问题

- 多个数据包在服务器收到数据包，导致tcp缓冲区，导致数据粘在一起了
- tcp并不会导致数据包的混乱，tcp会保证数据的准确性和顺序性
- 是我们逻辑缓冲区会造成粘包

### 原因

- 客户端的发送频率高于服务器的接收频率
- tcp底层的安全和效率不允许字节数特别少的小包发送频率过高
- 发送缓冲区有上次未发送的数据，或者接收缓冲区有上次未取的数据

### 办法

- 切包的方式处理
- 使用tlv协议（消息id+消息长度+消息内容）

## 1.变量

### 代码例子

```rust
use std::io;//导入

fn main() {
    println!("guess math!");//打印宏
    println!("猜一个数");
    let mut guess = String::new();
    io::stdin().read_line(&mut guess).expect("无法读取");
    println!("你猜的数是:{}", guess);
}
```

### 是否为可变类型

- 对比c++，c++也有const作为不可变类型，但多为标记
- rust则是在编译期间会检查是否为不可变量更改数值

#### 不可变(默认变量类型)

```rust
let a;
```

#### 可变

```rust
let mut a;//mut为immutable这个英文的缩写
```

### 类型

#### String

- 字符串

### 创建实例方法
- new(函数)
- 使用(变量类型)::new();完成创建
```rust
let mut guess=Strimg::new();
```
### 2.输入和输出（继续看1.的例子代码）
### 使用函数
- println!();
- std::io::stdin().read_line(&(是否可变类型名) (变量名)).expect("");
#### println()
- 打印一个宏
#### println("{}"，guess)
- 此处的花括号为占位符
- 会被替换为后面的变量名
#### read_line()
- 将用户的输入放到字符串里面
- 需要可变类型
- 使用引用获取
- 保证可以简单安全的使用引用
- 返回io::Result(枚举类型)，有两个类型，为ok，err
#### expect
- 返回的是Err中端程序，将字符串程序打印
- 返回ok将输入的数据返回
#### 读取
```rust
io::stdin().read_line(&mut guess).expect("");
```
#### 输出
```rust
println!("{}",guess);
```
