# 网络编程
## 伪闭包
- 昨天的handle_read和handle_write代码致使可能会两次delete this，造成报错
### 解决问题
- 把智能指针作为参数传入函数
- 引入lambda进行智能指针加一
### 智能指针管理Session
- 延长指针周期
- 解决方案：将shared_ptr<Session>作为参数传入回调函数，通过智能指针增加引用计数，确保对象在异步操作期间存活。
#### 头文件
```cpp
class Server
{
public:
	Server(io_context& ioc, short port);
	void ClearSession(string uuid);//清除Session
private:
	void start_accept();
	void handle_accept(shared_ptr<Session> new_session,const boost_error_code& error);
	io_context& _ioc;
	tcp::acceptor _acceptor;
	map<string, shared_ptr<Session>>_session;//管理Session
};
```
#### 实现文件
```cpp
Server::Server(io_context& ioc, short port) :_ioc(ioc), _acceptor(ioc, tcp::endpoint(tcp::v4(), port))
{
	cout << "Server starting" << endl;
	start_accept();
}

void Server::start_accept()
{

	shared_ptr<Session> new_session=make_shared<Session>(_ioc,this);
	_acceptor.async_accept(new_session->Socket(), std::bind(&Server::handle_accept, this, new_session,
		std::placeholders::_1));//加入this使其变为普通函数对象
	//shared_ptr不会随右括号而关闭
}

void Server::handle_accept(shared_ptr<Session> new_session, const boost_error_code& error)
{
	if (!error)
	{
		new_session->Start();
		_session.insert(make_pair(new_session->Get_uuid(), new_session));
	}
	else {
	}
	start_accept();
}

void Server::ClearSession(string uuid)
{
	_session.erase(uuid);
}
```

void Session::Start()
{
	memset(_data, 0, max_length);
	_socket.async_read_some(boost::asio::buffer(_data, max_length),
		bind(&Session::handle_read, this, std::placeholders::_1, std::placeholders::_2,shared_from_this()));
}
void Session::handle_read(const boost_error_code& error, size_t bytes_transferred, shared_ptr<Session> _self_shared)
{
	if (!error)
	{
		cout << "server receive data is " << _data << endl;
		boost::asio::async_write(_socket, boost::asio::buffer(_data, bytes_transferred),
			std::bind(&Session::handle_write, this, std::placeholders::_1,_self_shared));
	}
	else
		{
			cout << "have error in read,error code is " << error.value() << endl;
			_server->ClearSession(uuid);
		}
}
