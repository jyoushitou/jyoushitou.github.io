# web && http
## http
- HTTP 默认端口：80
- HTTPS 默认端口：443
### http连接
#### 非持久HTTP
- 最多只有一个对象在TCP连接上发送
- 下载多个对象需要多个TCP连接
- HTTP/1.0开始使用非持久连接
#### 持久HTTP
- 多个对象的TCP连接传输
- HTTP/1.1默认使用持久连接
### 请求报文
#### 表单输入
##### Post方式
- 网页通常包括表单输入
- 包含在实体中的输入提交到服务器
##### URL
- GET方式提交
- 输入通过请求行的URL字段上载
#### 获取方式
##### HTTP/1.0
- GET
- POST
- HEAD
##### HTTP/1.1
- GET,POST,HEAD
- PUT(提交)
- DELETE(删除)
### 响应报文
#### 结构
##### 状态行
- HTTP/1.1 200 OK\r\n
- 协议/版本号 状态码
##### 首部行
- 1. Connection: close
- 通用作用：本次请求响应完成后，立即关闭 TCP 连接，不复用这条连接做下次请求。
- 2. Date: 格林威治时间
- 通用作用：服务器生成响应报文的标准 GMT 时间，用来做缓存校验、报文时序判断。
- 3. Server: 服务器版本
- 通用作用：标识服务器软件名称 + 版本 + 运行系统，是服务指纹标识。
- 4. Last-Modified: 资源最后修改时间
- 通用作用：当前返回的资源本身最后一次被修改的时间，用于浏览器协商缓存。
- 5. Content-Length: 数值
- 通用作用：响应正文实体的字节长度，客户端靠这个知道要收多少数据，保证接收完整。
- 6. Content-Type: text/html
- 通用作用：声明响应内容的MIME 类型，告诉客户端该用什么格式解析（这里是 HTML 网页）。
##### 数据
##### PS:
- 得自己维护报文
- 使用ASCII交换，和代码解释
### http为无状态的协议

- 绝大部分的网站都带有cookie
- 使无状态的协议变成有状态的协议

- 在web站点有一个后端数据库

- 各种状态服务
## web缓存(代理服务器)
- 目标：不访问原始服务器，就满足客户的请求
### 优点
- 访问速度快
- 服务器端载荷变少
- 使用较弱的ICP也能够提供有效服务
## 条件GET
- 目标：如果缓存器中的对象拷贝是最新的，就不要发送对象
### 缓存器
- 在HTTP请求中指定缓存拷贝的日期
### 服务器
- 如果缓存拷贝陈旧，则响应没包含对象
# FTP(文件传输协议)
- 默认端口：21
- 明文传播
## 控制连接与数据连接分开
### 控制连接（Control Connection）
- 端口：默认 TCP 21
- 作用：传命令与响应（登录、列目录、上传 / 下载指令、状态码）
- 生命周期：整个会话一直存在，直到用户退出
### 数据连接（Data Connection）
- 端口：主动模式默认 TCP 20；被动模式用服务器随机高端口
- 作用：只传文件内容或目录列表
- 生命周期：按需临时建立，传完立即关闭
## FTP命令、响应
### 1. 登录类
- USER 用户名：提交登录用户名
- PASS 密码：提交登录密码
- QUIT：退出 FTP 会话，断开控制连接
### 2. 目录操作类
- PWD：查看当前工作目录
- CWD 路径：切换当前目录
- CDUP：返回上一级目录
- MKD 目录名：创建文件夹
- RMD 目录名：删除空文件夹
### 3. 文件列表与查看
- LIST：列出目录下文件详情（创建数据连接传列表）
- NLST：只列出文件名，无详细信息
### 4. 文件传输类
- RETR 文件名：下载文件（服务器→客户端）
- STOR 文件名：上传文件（客户端→服务器）
- DELE 文件名：删除文件
### 5. 传输模式（主动 / 被动）
- PORT：主动模式，客户端告知自己的数据端口
- PASV：被动模式，服务器返回随机高端口，客户端主动连
### 6. 格式 / 类型设置
- TYPE A：ASCII 文本模式（默认）
- TYPE I：二进制模式（传程序、图片、压缩包必须用）
### 三、FTP 响应码（3 位标准码，分类记忆）
- 1xx 预备状态（正在处理）
- 125：数据连接已打开，开始传输
- 150：文件状态正常，即将打开数据连接
- 2xx 成功完成
- 220：服务器就绪，可以登录
- 221：会话结束，关闭连接
- 226：数据传输完成，关闭数据连接
- 230：用户登录成功
- 3xx 需要进一步输入
- 331：用户名正确，需要输入密码
- 4xx 临时错误（可重试）
- 425：无法打开数据连接
- 426：连接关闭，传输中断
- 5xx 永久错误（操作失败）
- 530：登录失败（用户名 / 密码错）
- 550：文件 / 目录不存在、权限不足
# FTP VS http
- FTP 专为传文件，双连接、有状态、21 端口，适合上传下载大文件
- HTTP 专为看网页，单连接、无状态、80 端口，适合浏览和接口请求
