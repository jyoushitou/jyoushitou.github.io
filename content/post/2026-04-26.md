# 协议层次和服务模型
## 服务和服务访问点(SAP)
### 服务
- 底层实体提供他们之间的通信的能力
- 垂直的关系
### 原语
- 服务提供的形式
- 例如socket api
### 服务访问点
- 区分服务对象
### 套接字
- 用于区分
### SAP VS 端口
- 作用：
- SAP:相邻两层之间的逻辑标识
- 端口:同一台主机内，区分不同应用进程
- 范围:
- SAP:数据链路层、网络层等底层(层与层之间)
- 端口:传输层→应用层之间(软件与软件)
### 面向连接的服务
- TCP的三次握手
- 要为通讯做好准备(缓冲)
### 无连接的服务
- UDP
- 无需为通讯做准备
## 协议
- 水平
# 数据单元(DU)
- 服务数据单元SDU:
- 上层来的数据(加入控制信息)
- 只关心内容，不加头部
- 协议数据单元PDU:
- 本层的数据包
- 用来在对等层之间通信
## Header+Body(SDU)+=PDU
## 拆包
- 将SDU拆分开，组成多个PDU
## 粘包
- 将多个SDU组成一个PDU
## 结构化
### 优点
- 便于独立设计实现
- 便于采用新技术
- 便于交流和讨论
### 缺点
- 效率低
# Internet协议栈
## 应用层(报文(message))
- 为人类和应用提供对应的服务
- FTP,SMTP,HTTP,DNS
## 传输层(报文段(segment))
- 进程到进程的服务
- 把网络层的不可靠的服务转换成可靠的服务
- 区分进程
- TCP,UDP
## 网络层(分组packet)
- 原主机到目标主机的端到端的传输(E2E)
- IP,路由协议
### 作用
- 路由和转发
- 点对点协议PPP,802.11(wifi),Ethernet
## 链路层(帧(frame))
- 传输以帧为单位的数据
- 区分bit的帧的开始和结束
- 解决了分组的传输
## 物理层(位(bit))
- 传输帧的bit
### 发送端
- 上层的(frame)帧的bit转换成介质的物理信号的发送出去
### 接收端
- 将接收的介质的物理信号转换成原本的帧
## P2P - E2E
- P2P:
- 同层 ↔ 同层 聊天，交换的是本层 PDU
- E2E:
- 不管中间路由器、二层交换机
- 只负责两端软件 / 应用的数据直达
